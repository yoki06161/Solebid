# 브라우저에서 HTTPS 접속 가이드

## 🔒 자체 서명 인증서 경고 해결 방법

개발 환경에서 자체 서명(Self-Signed) SSL 인증서를 사용하면 브라우저에서 보안 경고가 표시됩니다. 이는 **정상적인 동작**이며, 다음 방법으로 해결할 수 있습니다.

## Chrome/Edge에서 접속하기

### 1단계: HTTPS URL 접속
브라우저 주소창에 다음 중 하나를 입력하세요:
```
https://localhost:3443
```

또는 HTTP로 접속하면 자동으로 HTTPS로 리다이렉트됩니다:
```
http://localhost:3000
```

### 2단계: 보안 경고 화면
다음과 같은 경고 화면이 표시됩니다:

```
⚠️ 연결이 비공개로 설정되어 있지 않습니다

공격자가 localhost에서 사용자의 정보(예: 비밀번호, 메시지, 신용카드 등)를 
도용하려고 시도할 수 있습니다.

NET::ERR_CERT_AUTHORITY_INVALID
```

### 3단계: 고급 옵션 클릭
1. **"고급"** 버튼을 클릭합니다
2. **"localhost(안전하지 않음)로 이동"** 링크를 클릭합니다

### 4단계: 접속 완료
이제 Solebid 애플리케이션이 로드됩니다!

---

## Firefox에서 접속하기

### 1단계: HTTPS URL 접속
```
https://localhost:3443
```

### 2단계: 보안 경고 화면
```
⚠️ 경고: 잠재적 보안 위험 발견

Firefox가 localhost에 대한 잠재적 보안 위협을 감지하여 계속 진행하지 않았습니다.
```

### 3단계: 예외 추가
1. **"고급..."** 버튼을 클릭합니다
2. **"위험을 감수하고 계속"** 버튼을 클릭합니다
3. **"예외 추가..."** 버튼을 클릭합니다
4. **"보안 예외 확인"** 버튼을 클릭합니다

### 4단계: 접속 완료
이제 Solebid 애플리케이션이 로드됩니다!

## 브라우저 캐시 문제 해결

만약 위 방법으로도 접속이 안 된다면, 브라우저 캐시를 삭제하세요:

### Chrome/Edge
1. `Ctrl + Shift + Delete` 키를 누릅니다
2. **"쿠키 및 기타 사이트 데이터"** 체크
3. **"캐시된 이미지 및 파일"** 체크
4. **"데이터 삭제"** 클릭
5. 브라우저를 완전히 종료하고 다시 시작

### Firefox
1. `Ctrl + Shift + Delete` 키를 누릅니다
2. **"쿠키"** 체크
3. **"캐시"** 체크
4. **"지금 삭제"** 클릭
5. 브라우저를 완전히 종료하고 다시 시작

### 시크릿/프라이빗 모드 사용
캐시 문제를 피하려면 시크릿 모드로 접속하세요:
- **Chrome/Edge**: `Ctrl + Shift + N`
- **Firefox**: `Ctrl + Shift + P`

## 인증서를 신뢰할 수 있는 인증서로 추가 (권장)

매번 경고를 무시하는 대신, 인증서를 시스템에 신뢰할 수 있는 인증서로 추가할 수 있습니다.

### Windows에서 인증서 추가

#### 1단계: 인증서 파일 찾기
```
C:\Users\[사용자명]\IdeaProjects\Solebid\nginx\ssl\cert.pem
```

#### 2단계: 인증서 설치
1. `cert.pem` 파일을 더블클릭합니다
2. **"인증서 설치..."** 버튼을 클릭합니다
3. **"현재 사용자"** 선택 → **"다음"**
4. **"모든 인증서를 다음 저장소에 저장"** 선택
5. **"찾아보기..."** 클릭 → **"신뢰할 수 있는 루트 인증 기관"** 선택
6. **"다음"** → **"마침"** 클릭
7. 보안 경고가 나타나면 **"예"** 클릭

#### 3단계: 브라우저 재시작
브라우저를 완전히 종료하고 다시 시작합니다.

#### 4단계: 접속 확인
이제 `https://localhost:3443`에 접속하면 경고 없이 바로 접속됩니다!

## 🎯 빠른 테스트 방법

### 방법 1: HTTP로 접속 (자동 리다이렉트)
```
http://localhost:3000
```
→ 자동으로 `https://localhost:3443`으로 리다이렉트됩니다

### 방법 2: HTTPS로 직접 접속
```
https://localhost:3443
```
→ 경고 화면에서 "고급" → "계속 진행" 클릭

### 방법 3: 시크릿 모드 사용
1. `Ctrl + Shift + N` (Chrome/Edge) 또는 `Ctrl + Shift + P` (Firefox)
2. `https://localhost:3443` 접속
3. 경고 무시하고 계속 진행

## ❓ 자주 묻는 질문

### Q1: 왜 경고가 표시되나요?
**A**: 개발 환경에서 사용하는 자체 서명 인증서는 공인 인증 기관(CA)의 서명이 없기 때문입니다. 프로덕션 환경에서는 Let's Encrypt 같은 공인 CA의 인증서를 사용하면 경고가 표시되지 않습니다.

### Q2: 이 경고를 무시해도 안전한가요?
**A**: 개발 환경(localhost)에서는 안전합니다. 하지만 공개 인터넷에서는 절대 경고를 무시하면 안 됩니다!

### Q3: 프로덕션에서는 어떻게 하나요?
**A**: 프로덕션 환경에서는 Let's Encrypt를 사용하여 무료 공인 인증서를 발급받으세요. 자세한 내용은 `docs/ssl-https-setup-guide.md`를 참고하세요.

### Q4: 인증서를 신뢰할 수 있는 인증서로 추가했는데도 경고가 나타나요
**A**: 다음을 확인하세요:
1. 브라우저를 완전히 종료하고 다시 시작했나요?
2. 인증서를 "신뢰할 수 있는 루트 인증 기관" 저장소에 추가했나요?
3. 브라우저 캐시를 삭제했나요?

### Q5: Chrome에서 "NET::ERR_CERT_COMMON_NAME_INVALID" 오류가 나타나요
**A**: 이는 인증서의 Common Name이 접속 URL과 일치하지 않을 때 발생합니다. `localhost`로 접속하고 있는지 확인하세요. `127.0.0.1`이나 다른 IP로 접속하면 이 오류가 발생할 수 있습니다.

## 🔍 추가 확인 방법

### HTTPS 연결 확인
브라우저 주소창에서 다음을 확인하세요:
- 🔒 자물쇠 아이콘이 표시됨
- `https://localhost:3443` URL 표시
- 자물쇠 아이콘을 클릭하면 인증서 정보 확인 가능

### 인증서 정보 확인
1. 주소창의 🔒 아이콘 클릭
2. **"인증서"** 또는 **"연결이 안전함"** 클릭
3. 인증서 세부 정보 확인:
   - 발급 대상: localhost
   - 발급자: Solebid Development CA
   - 유효 기간: 365일
